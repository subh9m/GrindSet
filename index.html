THE 5-STEP APPROACH

A systematic method to solve any coding interview problem

01

UNDERSTAND & CLARIFY

Ask targeted questions about input sizes, edge cases, constraints, and expected output format before writing code.

02

BRUTE-FORCE FIRST

Start with the simplest solution. Demonstrate understanding and create a baseline to optimize from.

03

ANALYZE CONSTRAINTS

Input size reveals required time complexity. Use this as your biggest hint for algorithm selection.

04

OPTIMIZE & SELECT

Look for memoization opportunities, hash map optimizations, two pointers, binary search, or greedy approaches.

05

CODE & TEST

Write clean code with meaningful names. Test normal cases, edge cases, and boundary conditions.

RED FLAGS TO WATCH FOR

  • "Optimize for time" — Usually means sacrifice space complexity
  • "In-place" — Means O(1) extra space only, no additional arrays
  • "Online algorithm" — Process data as it streams in, can't store all
  • "Real-time" — Suggests need for very efficient solution
  • "Multiple test cases" — Your solution will run many times, optimize!

FOUNDATIONAL ALGORITHMS

Master these patterns to solve 90% of interview problems

HASH MAP

WHEN TO USE

Need O(1) lookups, counting frequencies, or checking if elements exist. Perfect for problems requiring fast access to previously seen data.

KEYWORDS

count, frequency, unique, duplicate, seen before, complement, pair sum, anagram, group by

EXAMPLES: Two Sum, Group Anagrams, Subarray Sum Equals K

BINARY SEARCH

WHEN TO USE

Searching in sorted arrays or when finding specific value in sorted space. Also powerful for "search for the answer" problems with monotonic property.

KEYWORDS

sorted array, find target, logarithmic time, first/last occurrence, peak, minimum/maximum in range, rotated

EXAMPLES: Search in Rotated Array, Find Peak Element, Sqrt(x)

SLIDING WINDOW

WHEN TO USE

Problems involving contiguous subarrays/substrings with specific properties. Especially when you need min/max/optimal subarray.

KEYWORDS

subarray, substring, contiguous, window, maximum/minimum length, sum equals k, at most k, consecutive

EXAMPLES: Longest Substring Without Repeating, Minimum Window Substring

TWO POINTERS

WHEN TO USE

Problems on sorted arrays, finding pairs, or processing elements from both ends. Great for reducing O(n²) to O(n).

KEYWORDS

sorted array, pair, triplet, palindrome, opposite ends, meeting in middle, remove duplicates, in-place

EXAMPLES: 3Sum, Container With Most Water, Valid Palindrome

BFS / DFS

WHEN TO USE

BFS for shortest paths in unweighted graphs and level-order traversal. DFS for exploring all paths and tree traversals.

KEYWORDS

shortest path, level-order, traverse, connected components, cycle detection, all paths, maximum depth

EXAMPLES: Number of Islands, Level Order Traversal, Course Schedule

BACKTRACKING

WHEN TO USE

Finding all possible solutions, combinations, permutations. Make choices, recurse, then undo choices.

KEYWORDS

all combinations, all permutations, generate all, place queens, solve puzzle, word search, partition

EXAMPLES: N-Queens, Generate Parentheses, Combination Sum

DYNAMIC PROGRAMMING

WHEN TO USE

Optimization problems with overlapping subproblems and optimal substructure. When brute force repeats calculations.

KEYWORDS

maximum/minimum, count ways, optimal, best, overlapping subproblems, subsequence, ways to, longest

EXAMPLES: Coin Change, LCS, Edit Distance, House Robber

HEAP / PRIORITY QUEUE

WHEN TO USE

Need min/max element repeatedly or maintain sorted order while inserting/removing. Perfect for "top k" problems.

KEYWORDS

kth largest/smallest, top k, median, priority, merge sorted, closest, running median, k most frequent

EXAMPLES: Kth Largest Element, Merge K Sorted Lists, Find Median

MONOTONIC STACK/QUEUE

WHEN TO USE

Finding next greater/smaller element or maintaining elements in increasing/decreasing order efficiently.

KEYWORDS

next greater, next smaller, previous greater, largest rectangle, trapping water, stock span

EXAMPLES: Next Greater Element, Largest Rectangle in Histogram

PREFIX SUM

WHEN TO USE

Calculate range sums quickly or when subarray sums appear frequently. Preprocessing for O(1) queries.

KEYWORDS

range sum, subarray sum, sum between indices, continuous subarray, multiple queries

EXAMPLES: Range Sum Query, Subarray Sum Equals K

ADVANCED ALGORITHMS

Level up with these sophisticated techniques

TRIE (PREFIX TREE)

WHEN TO USE

String prefix matching, autocomplete systems, or efficient storage of strings with common prefixes.

KEYWORDS

prefix, autocomplete, word search, dictionary, common prefix, word break

EXAMPLES: Implement Trie, Word Search II, Longest Word

UNION-FIND (DSU)

WHEN TO USE

Dynamic connectivity problems, detecting cycles in undirected graphs, grouping elements into connected components.

KEYWORDS

connected components, union, find, cycle detection, dynamic connectivity, friends, groups

EXAMPLES: Number of Connected Components, Redundant Connection

SEGMENT TREE

WHEN TO USE

Range queries with updates on arrays. When both query and update operations need to be efficient.

KEYWORDS

range sum, range minimum/maximum, update range, interval queries, lazy propagation

EXAMPLES: Range Sum Query - Mutable, Count of Smaller Numbers

TOPOLOGICAL SORT

WHEN TO USE

Ordering tasks with dependencies, course scheduling, linear ordering of vertices in DAG.

KEYWORDS

dependencies, prerequisites, course schedule, directed acyclic graph, ordering

EXAMPLES: Course Schedule I & II, Alien Dictionary

DIJKSTRA'S ALGORITHM

WHEN TO USE

Finding shortest paths in weighted graphs with non-negative edge weights. Use priority queue for best performance.

KEYWORDS

shortest path, weighted graph, minimum cost, distance, network, routing

EXAMPLES: Network Delay Time, Cheapest Flights Within K Stops

GREEDY ALGORITHMS

WHEN TO USE

Optimization problems where locally optimal choice leads to globally optimal solution. Validate carefully.

KEYWORDS

maximum/minimum, optimal, greedy choice, activity selection, scheduling, interval

EXAMPLES: Jump Game, Gas Station, Non-overlapping Intervals

OPTIMIZATION WARNING SIGNS

  • Nested loops with same variable — Two pointers might work
  • Recursive calls with same parameters — Memoization needed
  • Searching in sorted data — Binary search opportunity
  • Finding extremes repeatedly — Heap/Priority queue
  • Counting occurrences — Hash map for O(1) lookups
  • Range queries — Segment tree or prefix sums

YOUR 8-WEEK ACTION PLAN

From foundation to expert level mastery

WEEK 1-2

Foundation Building

Master Hash Maps, Two Pointers, and Sliding Window. Practice 3-5 easy problems daily. Focus on understanding patterns over speed.

WEEK 3-4

Core Algorithms

Master BFS, DFS, Binary Search, and Basic DP. Practice 2-3 medium problems daily. Focus on recognizing when to use each algorithm.

WEEK 5-6

Advanced Patterns & Problem Types

Tackle Tries, DSU, Segment Trees, Monotonic structures, and advanced DP. Solve 1-2 hard problems every other day. Build short notes for pattern recognition.

WEEK 7

Mock Interviews & Timed Practice

Simulate interview conditions: 45–60 minute mock interviews, pair-programming or whiteboard style. Analyse mistakes, write clean solutions and discuss trade-offs.

WEEK 8

Refinement & Final Prep

Polish weak areas, revisit tricky problems, build a cheat-sheet of algorithms and edge cases. Do 2 full mock interviews and 3 timed contests to finish strong.

GRADUATION

Go Live

Apply consistently. Use your notes during interviews, reflect after each session, and iterate on feedback. Confidence and clarity beat rushing—you got this.
